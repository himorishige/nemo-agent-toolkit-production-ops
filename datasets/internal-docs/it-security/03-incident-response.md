# セキュリティインシデント対応手順

セキュリティインシデント発生時の通報・初動・恒久対応の手順です。

## インシデントの定義

以下のいずれかに該当する場合、本手順を発動します。

- 不正アクセス（社外からの侵入、社員アカウントの乗っ取り）
- データ漏えい（顧客情報、社員情報、知的財産）
- マルウェア感染（端末・サーバー問わず）
- フィッシング被害（クリック後にパスワード入力した場合）
- 物理的セキュリティ侵害（オフィス侵入、機器盗難）

## 通報経路

- 平日 9:00 〜 18:00: 情シス部の `it-incident@asahi-systems.example.com` に件名「インシデント報告」で通報
- 上記時間外: 緊急用 Slack `#incident-response` チャンネル + 当番制の電話番号（社内ポータルで確認）

## 初動（発覚から 1 時間以内）

1. 関連する端末・アカウントの隔離（情シス部）
2. 証跡保全（ログ、メモリダンプの取得）
3. インシデント担当者（CSIRT）の召集

## 恒久対応

- 影響範囲の特定（48 時間以内）
- 根本原因分析と再発防止策（1 週間以内）
- 必要に応じて顧客・規制当局への通知

## 演習

四半期に 1 回、机上演習を実施します。全社員の参加が推奨されます。
